# Book Factory Documentation

This folder contains the binding project documentation for Book Factory.

## Documentation Areas

- `architecture/` describes the software architecture and module boundaries.
- `adr/` contains Architecture Decision Records.
- `roadmap/` describes phases, milestones, and planned development order.
- `standards/` defines coding standards and naming conventions.
- `github/` describes repository and collaboration workflows.
- `documentation/` defines how documentation itself is organized.
- `product/` defines product vision, MVP scope, workflow, data model drafts, GPT directory concepts, quality checklists, and prompt handover.

## Documentation Rules

- Architecture decisions belong in ADRs.
- Module changes must be reflected in the module overview.
- Roadmap changes must be reflected in the roadmap documents.
- Documentation should be written clearly enough that a beginner can understand the next step.
