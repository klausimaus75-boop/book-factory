# GitHub Structure

## Repository Structure

The repository should support professional collaboration from the beginning.

## Branching

Recommended branch types:

- `main` for stable project state
- `codex/...` for Codex-assisted work
- `feature/...` for feature implementation
- `docs/...` for documentation-only work
- `fix/...` for bug fixes

## Issues

Issues should be used for:

- feature planning
- architecture tasks
- documentation tasks
- bugs
- technical debt

## Pull Requests

Pull requests should remain focused. A pull request should not mix unrelated architecture, documentation, and implementation work unless the relationship is clear.

## Labels

Recommended labels:

- `architecture`
- `documentation`
- `workflow`
- `module`
- `frontend`
- `storage`
- `quality-control`
- `technical-debt`
- `beginner-guidance`
