# 0004: Children's Book as First MVP Book Type

## Date

2026-07-10

## Status

Accepted

## Problem Statement

Book Factory needs a first concrete book type for the MVP. Without a specific book type, the first workflow would remain too abstract and the product definition would not be testable.

## Decision

The first MVP book type will be children's book.

## Rationale

Children's books exercise many important Book Factory capabilities: concept creation, target age handling, character consistency, plot structure, manuscript creation, illustration prompt planning, cover concept preparation, KDP metadata, and quality review.

This makes the workflow broad enough to validate the platform while still being specific enough to define clearly.

## Alternatives

- Start with coloring books.
- Start with puzzle books.
- Start with a generic book workflow.
- Start with KDP metadata only.

## Consequences

- The first workflow can be documented concretely.
- Quality checks can be age- and audience-specific.
- The platform must handle both text and visual planning outputs.
- Other book types remain future extensions and should not be built into the MVP prematurely.
