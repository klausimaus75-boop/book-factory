# Architecture Decision Records

This folder contains all Architecture Decision Records for Book Factory.

ADRs are binding project documentation. They explain important decisions, why they were made, which alternatives were considered, and what consequences follow.

## ADR Format

Each ADR must contain:

- Title
- Date
- Status
- Problem Statement
- Decision
- Rationale
- Alternatives
- Consequences

## Status Values

- Proposed
- Accepted
- Superseded
- Rejected

## Naming

ADR files use this format:

`0001-short-decision-title.md`

Numbers are sequential and must not be reused.
