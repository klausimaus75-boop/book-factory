# MVP Scope

## MVP Goal

The first MVP must guide a user from a children's book idea to a complete, structured children's book project.

The MVP is successful when a user can create a project, move through the defined workflow, generate handover prompts, use specialized GPTs manually, save returned results, run quality checks, and see project progress.

## Included in the MVP

The MVP must support:

- creating a new children's book project
- entering a project title
- defining the target audience
- defining the age range
- entering the book topic
- selecting the book format
- defining the page count
- defining the narrative perspective
- defining style and tone
- displaying workflow steps
- generating a complete handover prompt
- opening the matching GPT link
- pasting a GPT result back into the project
- saving the result
- reviewing the result
- unlocking the next workflow step
- displaying project progress

## First Book Type

The first supported book type is:

Children's book

This choice keeps the first workflow concrete while still exercising core platform concepts such as planning, characters, structure, manuscript, image prompts, cover concept, and KDP metadata.

## Not Included in the MVP

The MVP does not include:

- paid features
- user accounts
- team features
- direct OpenAI API usage
- automatic Amazon publishing
- image generation inside the application
- complex role management
- multi-user collaboration
- automated copyright or trademark checks
- advanced analytics

## MVP Boundary

The MVP should prove the production workflow, not the entire future product.

The most important test is whether the user can reliably continue from one production step to the next without losing context.

## Beginner Support Requirement

Every MVP step should clearly answer:

- what the user must enter
- why the information matters
- which GPT will be used
- what result is expected
- what must be checked before continuing
