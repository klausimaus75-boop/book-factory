# Development Phases

## Phase 0: Project Foundation

Goal: Create a professional basis before implementation.

Includes:

- repository structure
- architecture documentation
- module overview
- ADR system
- roadmap
- standards
- GitHub templates
- documentation structure

Status: Completed

## Phase 1: Product Definition

Goal: Define the first usable scope.

Includes:

- first target book type
- minimum project data model
- first workflow outline
- first GPT directory model
- first prompt template concept
- storage decision ADR
- frontend technology decision ADR

Status: In progress

## Phase 2: Technical Foundation

Goal: Set up the application framework without feature overload.

Includes:

- frontend app scaffold
- routing
- layout foundation
- design system foundation
- persistence prototype
- testing setup
- linting and formatting

## Phase 3: First Vertical Workflow

Goal: Implement one complete end-to-end workflow for one book type.

Includes:

- project creation
- book type selection
- workflow step display
- prompt generation
- GPT opening
- result intake
- manual quality check
- progress update

## Phase 4: Production Hardening

Goal: Make the first workflow reliable.

Includes:

- validation
- error states
- version history
- export flow
- backup concept
- responsive QA
- accessibility review

## Phase 5: Expansion

Goal: Add additional book types and reusable workflow patterns.

Includes:

- additional book type configurations
- reusable workflow templates
- advanced prompt library
- improved quality control
- import/export improvements
