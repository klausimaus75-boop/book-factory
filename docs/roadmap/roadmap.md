# Roadmap

## Immediate Priorities

1. Complete Phase 1 product definition for the children's book MVP.
2. Review and approve the first workflow.
3. Decide storage strategy.
4. Decide frontend technology stack.
5. Decide testing approach.
6. Only then start implementation.

## Near-Term Milestones

### Milestone 1: Foundation Complete

Status: Completed

Acceptance criteria:

- documentation structure exists
- ADR system exists
- initial ADRs exist
- module overview exists
- GitHub templates exist
- coding standards exist

### Milestone 2: First Product Scope

Status: In progress

Acceptance criteria:

- first target book type selected
- first workflow documented
- first data model drafted
- first GPT directory format drafted
- first quality checklist drafted
- prompt handover concept drafted
- MVP exclusions documented

### Milestone 3: Technical Stack Selected

Acceptance criteria:

- frontend framework selected
- storage strategy selected
- test strategy selected
- UI architecture selected
- ADRs created for each major decision

### Milestone 4: First Usable Prototype

Acceptance criteria:

- create a project
- select a book type
- view workflow steps
- generate a structured prompt
- open the correct GPT link
- paste result back into project
- mark a quality check result

## Long-Term Direction

Book Factory should become a reusable production platform that can support many book types and production methods without needing to rewrite the core application.
