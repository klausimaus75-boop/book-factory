# Documentation Structure

## Purpose

Book Factory documentation exists to make decisions, workflows, and module boundaries understandable before and during implementation.

## Folder Structure

```text
docs/
  adr/
  architecture/
  github/
  roadmap/
  standards/
  documentation/
```

## Document Types

### Architecture Documents

Describe the current intended architecture, module responsibilities, and system boundaries.

### ADRs

Record binding architecture decisions. ADRs should not be edited to rewrite history. If a decision changes, create a new ADR that supersedes the old one.

### Roadmap Documents

Describe project phases, priorities, milestones, and acceptance criteria.

### Standards Documents

Define coding, naming, testing, and documentation expectations.

### GitHub Documents

Define issue, pull request, branch, and collaboration workflows.

## Documentation Quality Bar

Documentation should be:

- specific
- beginner-friendly
- easy to update
- linked from related documents
- aligned with ADRs
